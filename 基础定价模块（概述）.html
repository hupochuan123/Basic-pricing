一：确认几个定义：


1：对于一本书籍或者漫画，基础的计费方式有3种：

按字收费（元／千字） 和 全本付费（买断某书籍）或者

按章收费（元／章）


2：对于书籍或漫画销售平台而言，默认4个平台：

Android 、iOS、H5、SDK

*平台数量不用写死，后续可能会增加



3:有3套 默认 基础定价【规则】，涵盖所有平台，我们称它

【规则1】、【规则2】、【规则3】

例：【规则1】

Android   --> 0.05元/千字 

iOS          --> 0.05元/千字                  

H5           --> 0.08元/千字                   

SDK         --> 0.08元/千字 

规则2

Android   --> 0.29元/章 

iOS          --> 0.29元/章                  

H5           --> 0.29元/章                   

SDK         --> 0.29元/章

规则3

Android   --> 全本付费

iOS          -->  全本付费             

H5           -->  全本付费                

SDK         -->  全本付费


                      


3：所有从书库同步过来的新书，都会带上定价规则。万一不带规则 ：小说默认规则1；漫画默认规则2；epub默认规则3；


4：不同【规则】之间属于并行结构，即同一批书同一时间只会遵循一套定价规则，时间最近的一次规则覆盖之前的规则。



4：【平台】字段由客户端在请求计费接口时（单章购买、批量购买）传给服务端，若无，则默认为 Android 平台价格。



二：基础定价模块几个主要功能



1：增加定价规则：


新建一个规则4，例如：


Android   --> 0.08元/千字

iOS          --> 0.08元/千字

H5           --> 0.12元/千字 

SDK         --> 0.10元/千字


*为防止影响线上业务，基础定价规则暂不提供运营人员删除功能



2：更改价格规则：

批量将一批书籍的定价规则改成另一个已存在的规则。

例如：

新建一个定价【规则4】，然后将一批在【规则1】的书籍改成【规则4】
